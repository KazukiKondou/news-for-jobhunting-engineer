## 目的

1. 公開中の全ページ・全リンクを**実際にブラウザで開いて**動作確認する
2. 不具合（404・内容不一致・表示崩れ・ナビ切れ・件数不整合）を発見したら**その場で修復し、push まで完了させる**

## Phase 1: 内部リンク・表示の実機検証

Claude in Chrome で https://news.kazukikondo.com/ を開き、以下を確認する:

- [ ] ホームのヒーロー・見出しリスト・「全文を読む」が最新日ページに正しく遷移する
- [ ] ホームの見出しリンク `/{日付}/#article-N` が該当記事カードまでスクロールする
      （記事IDのズレがないか。見出しのタイトルと遷移先カードのタイトルが一致すること）
- [ ] 「過去のニュース」の day-card 9枚がすべて開ける
- [ ] `/archive/` の全 day-card（54日分）が開ける。404 が 1 件もないこと
- [ ] 各日ページの day-nav（前の日 / 最新の日へ / 次の日）が正しく機能する
      （day-nav.js が /days.json で自己修復する設計。days.json が古い場合は再生成・再デプロイを検討）
- [ ] 件数表示の整合: day-meta の件数 = 見出し一覧の件数 = 記事カード数 = セクション件数の和
- [ ] 表示崩れ（空セクション、切れた日本語、文字化け）がないこと
- [ ] スマホ幅（375px）でもレイアウトが崩れないこと

## Phase 2: 外部リンク（元記事）の実機検証

効率のため2段階で行う:

1. **一括チェック**: 全日ページから記事URLを抽出し、curl で HTTP ステータスを取る
   ```bash
   grep -rhoP '<h3 class="article-card__title"><a href="\K[^"]+' site/*/index.html | sort -u
   ```
   これを `curl -s -o /dev/null -w '%{http_code}' -L -A 'Mozilla/5.0' <url>` で全件チェック。
2. **ブラウザ確認**: curl で 200 以外（404/403/000/リダイレクト）になったURLだけ、
   Claude in Chrome で実際に開いて最終判定する:
   - ページが開き、**記事タイトルと内容が一致** → OK
   - 404 / ドメイン消滅 / 全く別の内容 → **不具合**
   - 「募集終了」「公開期間終了」表示のインターン・イベントページ → **不具合**
   - ペイウォール・ログイン要求だが記事自体は実在 → OK（ただしメモに残す）

## Phase 3: 不具合の修復ルール

**絶対禁止: URLの推測・合成、ソースにない事実の追記（routine-prompt.txt ゼロ容認ルール）**

| 不具合 | 対処 |
|---|---|
| URLが404だが記事は実在 | WebSearch で正しいURLを探す。検索結果に出た実URLのみ差し替え可 |
| 記事の実在自体が確認できない | 記事ごと削除（件数より正確性） |
| 内容がタイトルと不一致 | 正しいソースが見つかれば差し替え、なければ削除 |
| 募集終了ページ | 同社の恒久URL（プレスリリース/採用トップ）に差し替え、なければ削除 |
| 内部リンク切れ・件数不整合 | 修正 |

### 記事を削除した場合の再生成ルール（1つでも漏れるとサイトが壊れる）

1. その日の `site/{日付}/index.html`:
   - article-N の ID を 1 から連番で振り直す
   - 見出し一覧（toc__list）を残存記事で作り直し、toc__summary-meta の件数を更新
   - day-meta の件数、各セクションの `articles-group__count` を更新
   - 記事が0件になったセクションはブロックごと削除（intro内の「今日は〜」文も削除）
   - day-intro を残存上位3記事のタイトル（ダッシュ以降切り落とし）で再生成
2. その日が0件になったら日ディレクトリごと削除し、他ページからの「前の日」リンクを
   直近の生存日に繋ぎ直す（`<link rel="prev">` も）。最古日になったページは前の日ボタン自体を削除
3. `site/index.html`: LATEST_DAY（最新日が変わった場合）と RECENT_DAYS の day-card
   （上位3タイトル・件数）を更新
4. `site/archive/index.html`: 該当 day-card の上位3タイトル・件数、hero の「N日分」を更新。
   archive の上位3タイトルはホーム見出し上位3と一致させる
5. 修正後に機械チェック: 全ページで「ID連番 = TOC件数 = day-meta = セクション和」
   「内部リンク先ディレクトリが実在」を確認するスクリプトを書いて回す

## Phase 4: 記録と公開

1. 検証ログを `logs/browser-check-$(date +%Y-%m-%d).json` に保存:
   `{url, http_status, browser_verdict(ok/fixed/deleted), action, reason}` の配列
2. コミット・プッシュ:
   ```bash
   git -c user.name='Kazuki Kondo' -c user.email='kizuka0621@gmail.com' \
     commit -am "fix: ブラウザ実機検証によるリンク切れ・不具合修復 ($(date +%Y-%m-%d))"
   git push origin main
   ```
3. デプロイ後（約5分）に、修正したページを Claude in Chrome でもう一度開き、
   修復が本番に反映されたことを**実機で再確認**する（これを省略しない）
4. 最後にサマリーを報告: 検証URL数 / OK / 修正 / 削除、残課題（ペイウォール等のメモ）

- 内部リンクの 404 が 0 件

- 件数・ID・見出しの不整合が 0 件
- push 済み + 本番反映をブラウザで再確認済み
