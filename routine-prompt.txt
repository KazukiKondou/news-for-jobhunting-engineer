# エンジニア就活ニュース 日次配信 routine (07:00 JST)

毎朝7時(JST)に発火。今日付の日次ニュースを2セクションで作成し、
サイト公開 + LINE配信 を実行する。

対象読者: **Web系・IT系事業会社** を志望する就活生(主に学部3〜4年・修士1〜2年)

「Web系・IT系事業会社」の定義:
- 自社プロダクトを持つWeb/SaaS/モバイル/プラットフォーム企業
- 例: メルカリ、LINEヤフー、DeNA、サイバーエージェント、リクルート、マネーフォワード、
       freee、Sansan、SmartHR、note、ラクスル、LayerX、Speee、ZOZO、ピクシブ等
- 外資テック: Google、Meta、Amazon、PayPay、Indeed 等
- 注目Web系スタートアップ全般

**対象外**(これらのニュースは原則扱わない):
- 大手SIer: NRI、NTTデータ、NEC、富士通、日立、TIS、SCSK 等のSI事業部
- 大手コンサル: アクセンチュア、デロイト、PwC 等
- 重厚長大企業のIT部門・グループ会社のSI部門
- 業務系COBOL、SI向け開発方法論
- IPA高度試験を採用条件として強調する話題(中立に扱う場合は可)

---

## 全体フロー

1. ニュース収集(2セクション)
2. site/YYYY-MM-DD/index.html 生成
3. site/index.html / site/archive/index.html 更新
4. GitHubへ commit + push
5. ssh kondo-home-server "/home/kondo/news/news-for-jobhunting-engineer/deploy.sh"
6. LINE push API で本日サイトURL + 見出し一覧を配信
7. https://news.kazukikondo.com/YYYY-MM-DD/ の 200 確認

## セクション 1 — IT就活ニュース (5件固定、下限3件・上限5件)

以下のジャンルから多様にミックス。同じジャンルだけに偏らないこと。

- **採用ニュース**: Web系メガ・スタートアップの採用枠/初任給/応募開始/早期化
- **企業動向**: 注目Web/SaaS企業の業績/組織変更/M&A、外資テックの日本拠点動向
- **スタートアップ**: シリーズA〜後期の資金調達、IPO、新サービスローンチ
- **技術トレンド**: Web系で実務利用される技術 (React/Vue/Next.js/TypeScript/Go/Rust/
  クラウド/DB/OSS) の主要リリース、注目OSS、面接ネタになる技術トピック
- **カンファレンス・イベント**: 国内外のWeb系技術カンファ、採用イベント、ハッカソン
- **キャリア・働き方**: Web系エンジニアの年収レポート、フリーランス、副業/リモート、
  転職事情、外資・海外採用

選定基準:
- 「Web系/事業会社を志望する就活生に効くか」を最優先
- AI関連は relevance がある時のみ含める(セクション独立化はしない)
- ソースは各社公式ブログ/Publickey/ITmedia/TechCrunch Japan/Findy/Forkwell/メルカリTech 等
- **大手SIer/コンサル/重厚長大IT部門のニュースは含めない**

要約の禁止語彙(SIer・コンサルPR臭):
- "DXパートナー" / "DX推進" / "経産省連携" / "業務改革支援"
- "ワンストップ" / "ソリューション提供" / "全社最適"
- 「Web系プロダクト/技術/採用」の視点で語る。コンサル/SIer向けの言い回しを使わない。

要約の禁止表現(メタコメント・読者呼びかけ):
- 「面接で頻出になりそう」「ESネタになる」「ESに書ける」など、書き手の評価
- 「就活生は注目」「押さえておきたい」「要チェック」「注目したい」「見逃せない」「知っておきたい」など、読者への呼びかけ
- 事実(何が起きたか + 数字 + なぜ嬉しいか)だけを記述。メタ評価はタグや別フィールドで表現。

セクション 1 内のジャンルバランス(5件選定時のhard cap、6件以上に膨らませない):
- 採用ニュース(枠拡大/初任給/応募開始等)は最大2件まで
- 技術トレンド(OSSリリース/言語/フレームワーク)は最低1件は入れる
- 同じ企業の複数記事は不可(1社1記事まで)
- 1記事が複数ジャンルに該当する場合、より上位の主カテゴリで1カウント
- 採用枠ニュースは国内学部・修士からエントリー可能なものを優先(海外限定の採用イベントは対象外 or 視野を広げる側に回す)

表示順ルール(daily / home共通、5件):
- (1) 採用ニュース ×最大2 → (2) 技術トレンド → (3) 資金調達 or 企業動向 → (4) キャリア/年収
- 採用2本に技術トレンドが続くリズムを基本に、金額ニュース(調達・年収)を中盤、キャリア軸を末尾
- 6件目以降(空き在庫)は翌日繰越

同日内・数値カニバリ単一ルール:
- 主役フックは1記事につき1個(最も差別化される数値・固有名詞)
- 同日5件のうち、同一の単位ヒット(万円/億円/%/倍/名/円)が3件以上に集中したら、最も重複する1件を別軸の数値(期日/対象範囲/順位/拠点数/期間)に強制差し替え
- 差し替え不能なら翌日繰越
- 例: 「720万円」「850万円」「500名」が混在しているが、もし「+40%」「+20%」「+8%」と % が3件以上になったら、うち1件は別軸数値に振り替える

技術リリースの要約フォーマット:
- 「何が変わったか」「数値で嬉しさ」の2点に絞る
- バージョン番号は正確に。Vue/React/Next等は実在の機能名・廃止履歴を踏まえる(誤情報を絶対に書かない)

## セクション 2 — 視野を広げる (1〜3件、IT就活が5件しっかり詰まる日は1件・薄い日は2〜3件と相関で調整)

冒頭に必ず以下のリード文を表示:
> 視野を広げるためのニュース。IT就活生は技術に偏りがちなので、ここではあえてIT以外の話題も入れています。

ジャンル不問:
- 文系: 経済/社会/政治/文化/教育/哲学/歴史
- スポーツ
- エンタメ・芸術・カルチャー
- 科学(医療/生物/宇宙/環境/物理など、IT外)
- 国際情勢
- 環境・社会問題

選定基準:
- 話題性は不問(マイナーでもOK)
- 「知って何かの引き出しになる」「就活面接の雑談・教養として効く」を基準
- IT関連は絶対に入れない
- **直近1週間の一般紙1面級ニュース(誰でも知ってる超メジャー話題)は原則回避**
- 2件入れるなら、ジャンルは異なる軸から(例: 科学×文化、環境×国際、スポーツ×経済)
- 各ジャンルローテーション目安: 科学/環境/国際情勢/文化・芸術/経済/社会 から

## HTML 構造

`templates/day.html` をベースに、`{{ARTICLES}}` を以下の構造に差し替え:

```html
<section class="articles-group">
  <div class="articles-group__header">
    <h2 class="articles-group__title">
      IT就活ニュース
      <span class="articles-group__count">5件</span>
    </h2>
  </div>
  <section class="articles">
    <article class="article-card" id="article-1">...</article>
    ...
  </section>
</section>

<section class="articles-group articles-group--other">
  <div class="articles-group__header">
    <h2 class="articles-group__title">
      視野を広げる
      <span class="articles-group__count">2件</span>
    </h2>
  </div>
  <p class="articles-group__intro">
    視野を広げるためのニュース。IT就活生は技術に偏りがちなので、ここではあえてIT以外の話題も入れています。
  </p>
  <section class="articles">
    <article class="article-card" id="article-6">...</article>
    ...
  </section>
</section>
```

各 article-card 内構造:
- tags → h3(title link) → p(summary 3-4行) → meta(source + 元記事リンク)
- id は通し番号(全セクション通して article-1, article-2, ...)
- summary は 3-4行程度。冗長な前置きは避け、要点→補足の順
- day-intro は最大80字。具体名詞か数字を最低1つ含める。
  - **禁止**: 件数 / セクション名 / 「押さえられる」等のメタ語
  - **OK例**: 「初任給720万・採用枠500名・Vue 3.6 が同日に並んだ。」
  - **NG例**: 「採用と技術の節目が重なった一日。N件です。」(メタ語のみ)
- 記号: `<` `>` `&` は文字実体参照、code片は `<code>...</code>`

## index.html / archive.html 更新

### index.html
- `<!-- LATEST_DAY_START -->` 〜 `<!-- LATEST_DAY_END -->`:
  - hero に当日 date / intro / N件のニュース
  - その下に **2セクション分の「見出し一覧」のみ** を表示(article-card本体は転載しない)
    - 各 `<li>` は: `<span class="num">NN</span> <span class="title">タイトル</span> <span class="sub">15〜22字のサブテキスト</span> <span class="source">ソース名(≤12字)</span>`
    - **サブテキストは記事の数値・固有名詞をフックに**(例: 「中央値720万円、前年比+8%」「Vapor Modeが安定化、大規模レンダリング約2倍」)
    - **<strong>は1記事1箇所、数値または固有名詞のみ。動詞・状態語(「急増」「新設」「強化」)には使わない**
    - ホームだけ見て離脱したユーザーに数字を渡す。モバイル幅375pxで折り返さない長さに抑える
    - source 名は会社名+「公式」程度に省略 (「LINEヤフー プレスリリース」→「LINEヤフー公式」)
    - 各見出しは titleクリックで `/{DATE}/#article-N` へジャンプ
    - 「全文を読む →」CTAボタンで `/{DATE}/` へ
  - 末尾に day-nav (前日リンク + 「最新の記事です」status)
  - ★article-card本文は日付ページ(/{DATE}/)に専有。ホームに転載しない(canonical衝突防止・カニバリ防止)
- `<!-- RECENT_DAYS_START -->` 〜 `<!-- RECENT_DAYS_END -->`:
  - 前日 day-card を先頭追加、計9枚まで保持
  - 「過去のニュース」セクション title は2件以上で「直近 N 日分」、1件なら「前日のニュース」

### archive/index.html
- `<!-- ARCHIVE_LIST_START -->` 〜 `<!-- ARCHIVE_LIST_END -->`:
  - 月でグループ化 (`<div class="section__group">`)、当日カードを月セクション先頭に追加
  - **当日カードの3件抜粋は home の `.home-headlines-list__title` テキストをそのまま流用** (daily/index.html の長尺h3は使わない。差し替え時は両側更新、整合チェック必須)
  - hero の「合計 N 日分のニュース」のカウント更新

## デプロイ

```sh
cd /tmp/news-site
rm -rf .git
git clone git@github.com-news:KazukiKondou/news-for-jobhunting-engineer.git . 2>/dev/null \
  || (git fetch origin && git reset --hard origin/main)

# (ファイル書き換え)

git add -A
git -c user.name='Kazuki Kondo' -c user.email='kizuka0621@gmail.com' \
  commit -m "news: $(date +%Y-%m-%d) のニュースを追加"
git push origin main

ssh kondo-home-server "bash /home/kondo/news/news-for-jobhunting-engineer/deploy.sh"
sleep 4
curl -sf "https://news.kazukikondo.com/$(date +%Y-%m-%d)/" > /dev/null \
  || (echo "ERROR: public URL not 200" >&2; exit 1)

# Home <-> Archive title sync check (deploy前にローカルで実行)
python3 -c '
import re, sys
home = open("site/index.html").read()
arch = open("site/archive/index.html").read()
home_titles = re.findall(r"home-headlines-list__title">([^<]+)</span>", home)[:3]
# archive 当日カード(冒頭=月の先頭)から li 3件
m = re.search(r"<a class=\"day-card\"[^>]+href=\"/[^/]+/\".*?<ul[^>]*>(.*?)</ul>", arch, flags=re.DOTALL)
arch_titles = re.findall(r"<li(?:\s+class=[^>]+)?>(?!<)([^<]+)</li>", m.group(1)) if m else []
arch_titles = [t for t in arch_titles if not t.startswith("他 ")]
missing = [t for t in home_titles if t not in arch_titles]
if missing:
    print(f"ERROR: archive day-card missing home titles: {missing}", file=sys.stderr); sys.exit(1)
' || (echo "ERROR: home/archive title sync mismatch" >&2; exit 1)
```

## LINE 配信

LINE channel access token は環境変数 `LINE_TOKEN`、送信先IDは `LINE_TO` から取得。
LINEはスマホ縦1スクロールで全部見える長さに圧縮する。絵文字は使わない。

```sh
DATE_SLASH="$(date +%-m/%-d)"
TOTAL=7  # 件数
URL="https://news.kazukikondo.com/"

MSG=$(cat <<EOF
${DATE_SLASH} ニュース${TOTAL}件 → ${URL}
【IT就活】
・<title ≤22字> — <数値フック ≤14字>
・...(5件)
【視野】
・<title ≤22字>(数値フック任意)
EOF
)

curl -sS -X POST https://api.line.me/v2/bot/message/push \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LINE_TOKEN" \
  -d "$(jq -n --arg to "$LINE_TO" --arg text "$MSG" \
        '{to: $to, messages: [{type: "text", text: $text}]}')"
```

1メッセージ。記事個別リンクは出さない(サイトで見てもらう設計)。
空行は最小限。【見出し】の前後にも入れない。

LINEタイトル短縮ルール(最大28文字、超えたら以下の優先順で削る):
1. 副題ダッシュ「— 以降」を全削除
2. 冗長な動詞を削る:「リリース/発表/公開/開始」など(文脈で意味が明らかな場合)
3. 補足修飾を削る:「エンジニア」「新卒」「向け」など (記事の核と関係薄い修飾語)
4. 一般語を簡略化:「拡大」→省略可、「改定」→省略可、「実施」→省略可
- 数字(年・人数・金額・%)は絶対に残す
- 削っても28字を超える場合のみ文末「…」省略
- 元タイトル → 短縮例:
  - 「LINEヤフー、2027年新卒エンジニア採用枠を500名に拡大」→「LINEヤフー、新卒採用枠を500名に — 前年比+40%」
  - 「IEA、2025年に世界の太陽光発電が初めて石炭を上回ると予測」→「IEA、2025年に太陽光が石炭超え予測」
- 数値フック: home の `.home-headlines-list__sub` の中核数字を再利用。フックが無い記事はダッシュ以降を省略。

## 失敗時の挙動

- ニュース不足:
  - IT就活ニュース最低3件、視野を広げる最低1件で許容(下回ったら異常終了 & LINE通知)
- deploy失敗: LINE配信せずに異常終了
- LINE失敗: サイト更新は維持(ログに残してretry不要)
- HTML生成失敗時: 既存日付ページに上書きしない(git push前にチェック)
